Create `internal/chat/mediator.go` in package `chat`.

Define `Message` struct for stored messages:
```go
type Message struct {
    From    string `json:"from"`
    To      string `json:"to"`
    Content string `json:"content"`
}
```

Define `ChatMediator` interface (per TASKS.md):
```go
type ChatMediator interface {
    SendMessage(from, to, message string)
    AddParticipant(participant ChatParticipant)
}
```

Define `ChatParticipant` interface (per TASKS.md):
```go
type ChatParticipant interface {
    GetName() string
    Receive(from, message string)
}
```

Implement `SupportChatMediator` struct:
- Fields: `participants map[string]ChatParticipant`, `history []Message`
- `NewSupportChatMediator() *SupportChatMediator` constructor initializes map and empty history slice.
- `AddParticipant(p ChatParticipant)` stores participant by `p.GetName()` key.
- `SendMessage(from, to, message string)`:
  1. Append Message{From: from, To: to, Content: message} to history.
  2. Look up recipient by `to` key in participants map.
  3. If found, call `recipient.Receive(from, message)`.
  4. If not found, silently skip (mediator handles routing; missing recipient is a no-op).
- `GetHistory(participant string) []Message` returns all messages where From == participant OR To == participant.

Implement `Customer` struct:
- Fields: `name string`, `mediator ChatMediator`, `received []Message` (messages this customer received).
- `NewCustomer(name string, mediator ChatMediator) *Customer` constructor. Sets mediator field.
- `GetName() string` returns name.
- `Receive(from, message string)` appends to `received` slice.
- `GetReceived() []Message` returns received messages (for testing/handler use).

Implement `Manager` struct (same structure as Customer -- separate type to demonstrate the pattern clearly):
- Fields: `name string`, `mediator ChatMediator`, `received []Message`.
- `NewManager(name string, mediator ChatMediator) *Manager` constructor.
- `GetName() string`, `Receive(from, message string)`, `GetReceived() []Message`.

CRITICAL per SRCH-06: Customer and Manager hold a `mediator ChatMediator` reference but NEVER hold a reference to another participant. The mediator's participants map is the only place where participant references exist together.

Follow conventions: Ukrainian doc comments, PascalCase exports, single-letter receivers (m for mediator, c for customer, g for manager). All types and interfaces in same package (per Pitfall 12 -- avoid circular imports).
