Create internal/order/validator.go with:

1. OrderRequest struct — the data that flows through the chain:
```go
type OrderRequest struct {
    Items   []string
    Address string
    Amount  float64
}
```

2. OrderValidator interface:
```go
type OrderValidator interface {
    SetNext(validator OrderValidator) OrderValidator
    Validate(req *OrderRequest) error
}
```
SetNext returns OrderValidator to allow fluent chaining: `stock.SetNext(address).SetNext(payment)`.

3. BaseValidator (embedded by concrete validators to inherit SetNext and PassToNext):
```go
type BaseValidator struct {
    next OrderValidator
}
```
- `func (b *BaseValidator) SetNext(v OrderValidator) OrderValidator` sets b.next = v, returns v (so chaining works).
- `func (b *BaseValidator) passToNext(req *OrderRequest) error` — if b.next != nil, calls b.next.Validate(req); otherwise returns nil (end of chain = validation passed).

4. StockValidator — checks that no item is "out-of-stock-item":
```go
type StockValidator struct {
    BaseValidator
}
```
- `func (v *StockValidator) Validate(req *OrderRequest) error` — iterates req.Items; if any item == "out-of-stock-item", returns `fmt.Errorf("stock validation failed: item %q is out of stock", item)`. Otherwise calls `v.passToNext(req)`.

5. AddressValidator — checks that address is not empty:
```go
type AddressValidator struct {
    BaseValidator
}
```
- `func (v *AddressValidator) Validate(req *OrderRequest) error` — if `strings.TrimSpace(req.Address) == ""`, returns `fmt.Errorf("address validation failed: address is required")`. Otherwise calls `v.passToNext(req)`.

6. PaymentValidator — checks that amount is > 0:
```go
type PaymentValidator struct {
    BaseValidator
}
```
- `func (v *PaymentValidator) Validate(req *OrderRequest) error` — if `req.Amount <= 0`, returns `fmt.Errorf("payment validation failed: amount must be greater than 0")`. Otherwise calls `v.passToNext(req)`.

7. Helper constructor to build the standard chain:
```go
func NewValidationChain() OrderValidator {
    stock := &StockValidator{}
    address := &AddressValidator{}
    payment := &PaymentValidator{}
    stock.SetNext(address).SetNext(payment)
    return stock
}
```

Follow conventions: receiver `v` for validators, `b` for BaseValidator. Export all types. Document each type with pattern name comment. Use fmt.Errorf for errors, import "strings" for TrimSpace.
