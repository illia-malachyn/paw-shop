2. Five concrete state types, each as a struct implementing OrderState:

- NewState: Name() returns "new". Next() sets order.state to &ConfirmedState{} and order.Status to "confirmed", returns nil. Cancel() sets order.state to &CancelledState{} and order.Status to "cancelled", returns nil.

- ConfirmedState: Name() returns "confirmed". Next() sets order.state to &ShippedState{} and order.Status to "shipped", returns nil. Cancel() sets order.state to &CancelledState{} and order.Status to "cancelled", returns nil.

- ShippedState: Name() returns "shipped". Next() sets order.state to &DeliveredState{} and order.Status to "delivered", returns nil. Cancel() returns fmt.Errorf("cannot cancel order: already shipped").

- DeliveredState: Name() returns "delivered". Next() returns fmt.Errorf("order already delivered"). Cancel() returns fmt.Errorf("cannot cancel order: already delivered").

- CancelledState: Name() returns "cancelled". Next() returns fmt.Errorf("order is cancelled"). Cancel() returns fmt.Errorf("order is already cancelled").

Create internal/order/iterator.go with:

1. OrderIterator interface:
```go
type OrderIterator interface {
    HasNext() bool
    Next() *Order
}
```

2. OrderCollection struct (holds the order store):
```go
type OrderCollection struct {
    orders []*Order
}
```
- Constructor: `func NewOrderCollection() *OrderCollection` returning empty collection.
- Method: `func (c *OrderCollection) Add(order *Order)` appends to orders slice.
- Method: `func (c *OrderCollection) CreateIterator() OrderIterator` returns &AllIterator{orders: c.orders, index: 0}.
- Method: `func (c *OrderCollection) CreateFilteredIterator(status string) OrderIterator` returns a FilteredIterator that yields only orders where order.GetState().Name() == status.
- Method: `func (c *OrderCollection) GetByID(id string) (*Order, bool)` — iterates orders, returns matching order or false.
- Method: `func (c *OrderCollection) Count() int` returns len(c.orders).

3. AllIterator (unexported struct):
```go
type allIterator struct {
    orders []*Order
    index  int
}
```
- HasNext() returns it.index < len(it.orders)
- Next() returns it.orders[it.index] and increments index. Returns nil if no next.

4. FilteredIterator (unexported struct):
```go
type filteredIterator struct {
    orders []*Order
    status string
    index  int
}
```
- HasNext() scans from current index forward to find next order with matching status (order.GetState().Name() == status), updates index to that position, returns true. If none found, returns false.
- Next() returns the current order and increments index past it.

Use index-based cursor approach per PITFALLS.md — NOT channels, NOT goroutines. The internal data structure (orders slice) is NOT exported; iterator is the access path.

Follow conventions: unexported iterator structs (allIterator, filteredIterator), exported interface and collection. Receiver `c` for collection, `it` for iterators.
