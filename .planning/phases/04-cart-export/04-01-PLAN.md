  <behavior>
    - Cart.AddItem adds a new item or increments quantity if product_id already exists
    - Cart.RemoveItem removes item by product_id; returns error if not found
    - Cart.UpdateQuantity changes quantity for existing product_id; returns error if not found; removes item if quantity <= 0
    - Cart.Save returns CartMemento with deep-copied items slice (not shared backing array)
    - Cart.Restore replaces cart items with deep copy from memento (not shared backing array)
    - CartHistory.Push appends memento to stack
    - CartHistory.Pop returns most recent memento and removes it; returns false if empty
    - Deep copy proof: Save, mutate cart, Restore from saved memento — cart matches saved state, not mutated state
  </behavior>
  <action>
    Create `internal/cart/cart.go` in package `cart`.

    Define types (per TASKS.md spec exactly):
    ```go
    type CartItem struct {
        ProductID string  `json:"product_id"`
        Name      string  `json:"name"`
        Price     float64 `json:"price"`
        Quantity  int     `json:"quantity"`
    }
    ```

    Cart struct with exported Items field `Items []CartItem` (needed for JSON serialization and Visitor access).

    Methods on *Cart:
    - `AddItem(item CartItem)` — if item with same ProductID exists, increment its Quantity by item.Quantity; otherwise append.
    - `RemoveItem(productID string) error` — find and remove by ProductID; return `fmt.Errorf` if not found.
    - `UpdateQuantity(productID string, quantity int) error` — if quantity <= 0, remove item; else update. Error if not found.
    - `Save() CartMemento` — create CartMemento with deep-copied slice: `snapshot := make([]CartItem, len(c.Items))` then `copy(snapshot, c.Items)`. Return value type (not pointer) per STACK.md recommendation.
    - `Restore(m CartMemento)` — deep copy from memento back: `c.Items = make([]CartItem, len(m.items))` then `copy(c.Items, m.items)`.

    CartMemento struct with UNEXPORTED field `items []CartItem` (opaque snapshot per Memento pattern — caretaker cannot access internals).

    CartHistory struct with unexported `stack []CartMemento`:
    - `Push(m CartMemento)` — append to stack.
    - `Pop() (CartMemento, bool)` — return last + true, or zero value + false if empty.

    CRITICAL per PITFALLS.md Pitfall 2: Both Save() and Restore() MUST use `make` + `copy`, never slice assignment. Add comment `// deep copy — not assignment` above each copy block. CartItem contains only value types (string, float64, int) so element-level deep copy is not needed — slice copy is sufficient.

    Follow existing conventions: package comment in Ukrainian, exported type docs in Ukrainian, interface at top of file, concrete types below.
