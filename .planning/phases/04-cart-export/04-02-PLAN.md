    ```go
    // Distinct method names per element type (Go does not support method overloading).
    type OrderVisitor interface {
        VisitItem(item cart.CartItem)
        VisitCart(c cart.Cart)
        Result() string
    }
    ```

    Implement concrete element wrappers so cart types can participate in Visitor without modifying the cart package:
    - `CartItemElement` struct wrapping `cart.CartItem`, Accept calls `visitor.VisitItem(e.Item)`
    - `CartElement` struct wrapping `cart.Cart`, Accept calls `visitor.VisitCart(e.Cart)` ã and also iterates items calling `visitor.VisitItem` for each item before `visitor.VisitCart`

    Implement `JSONExportVisitor`:
    - Holds `items []jsonItem` (unexported helper struct with ProductID, Name, Price, Quantity, Subtotal fields) and `total float64`.
    - `VisitItem`: appends item to items slice, adds price*quantity to total.
    - `VisitCart`: sets total (or already accumulated from items). Can be a no-op if total accumulated in VisitItem.
    - `Result()`: marshals struct `{"items": [...], "total": N}` via `encoding/json` with `json.MarshalIndent` for readability.

    Implement `TextReceiptVisitor`:
    - Holds `lines []string` and `total float64`.
    - `VisitItem`: appends line like `"Name - 2 x 100.00 = 200.00"`, adds to total.
    - `VisitCart`: appends separator and total line.
    - `Result()`: joins lines with newlines. Format:
      ```
      === Receipt ===
      Food A - 2 x 100.00 = 200.00
      Food B - 1 x 50.00 = 50.00
      ================
      Total: 250.00
      ```

    Add a convenience function `ExportCart(c cart.Cart, v OrderVisitor) string` that iterates cart.Items calling VisitItem for each, then calls VisitCart, then returns Result(). This is the primary entry point for handlers.

    Follow existing conventions: Ukrainian package/type comments, interface at top of file.
